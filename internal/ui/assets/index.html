Database Connection Setup

How It Works

This application compares the schemas of two databases and generates SQL scripts to synchronize the structure of the Target DB to match the Source DB.

  • Source DSN (Desired State): Your new reference database (e.g., local/development database with the most up-to-date schema).
  • Target DSN (Current State): The database you want to update (e.g., staging/production database).

Visual Comparison Details

Schema Synchronization Direction

The system detects differences to modify the Target Database (Current State) schema so that it follows/matches the Source Database (Desired State). The generated DDL queries will execute additions, type changes, or deletions on the Target Database.

Target DB (Current) ──► Source DB (Desired)
